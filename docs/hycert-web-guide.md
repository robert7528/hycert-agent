## 1. 工具箱

工具箱提供 6 個憑證常用工具，可直接貼上或上傳檔案操作，**不會存入系統**。適合臨時檢查、格式轉換等用途。若需要長期管理和自動部署，請使用「憑證管理」匯入。

### 1.1 檢核（Verify）

驗證憑證是否有效，檢查憑證鏈、到期日、金鑰配對。

**操作步驟：**
1. 貼上或上傳憑證（支援 PEM、DER、PFX、JKS、P7B）
2. 如有私鑰可一併貼上（驗證金鑰配對）
3. 如為 PFX/JKS 格式需輸入密碼
4. 按「執行」

**輸出資訊：**
- 主體資訊（CN、組織、國家）
- 簽發者
- 有效期間（起訖日、剩餘天數）
- SAN 清單
- 金鑰資訊（演算法、位元數）
- 指紋（SHA-256、SHA-1）
- 鏈驗證結果（是否完整、是否受信任）
- 警告訊息（如有）

### 1.2 解析（Parse）

解析憑證檔案，列出所有包含的憑證資訊。

**操作步驟：**
1. 貼上或上傳憑證檔案
2. 按「執行」

**輸出資訊：**
- 每張憑證的角色（伺服器 / 中繼 / 根）
- CN、簽發者、序號
- 有效期間、演算法
- SAN 清單

### 1.3 轉換（Convert）

將憑證從一種格式轉換為另一種格式。

**支援格式：**

| 來源 | 可轉換為 |
|------|---------|
| PEM | DER、PFX、JKS、P7B |
| DER | PEM、PFX、JKS |
| PFX | PEM、DER、JKS |
| JKS | PEM、DER、PFX |

**注意事項：**
- 轉換為 PFX / JKS 需要私鑰 + 密碼
- P7B 僅包含憑證，不含私鑰
- 可選擇是否包含憑證鏈

### 1.4 合併 Chain（Merge Chain）

將多個憑證檔案合併成一個完整的憑證鏈。

**操作步驟：**
1. 拖曳上傳多個憑證檔案，或手動貼上 PEM
2. 按「執行」
3. 系統自動排序：伺服器憑證 → 中繼憑證 → 根憑證

**輸出：**
- 合併後的 fullchain.pem（可複製或下載）

### 1.5 私鑰解密（Decrypt Key）

將加密的私鑰轉為明文，適用於 Nginx / Apache 不輸入密碼啟動。

**操作步驟：**
1. 貼上加密的私鑰
2. 輸入密碼
3. 按「執行」

**輸出：**
- 解密後的明文私鑰（PEM 格式）
- 金鑰類型與位元數

### 1.6 產生 CSR（Generate CSR）

產生憑證簽署請求（CSR）與私鑰。

**必填欄位：**
- 網域名稱（CN），支援萬用字元（如 `*.example.com`）

**選填欄位：**
- SAN（多個域名，逗號分隔）
- 組織（O）、部門（OU）、國家（C）、州/省（ST）、城市（L）
- 金鑰類型：RSA（2048/4096）或 EC（P-256/P-384）
- 私鑰密碼（選填）

**輸出：**
- CSR 檔案（可下載）
- 私鑰檔案（可下載，請妥善保管）

## 2. 憑證管理

管理所有已匯入的憑證，包含查詢、匯入、檢視、編輯、下載、刪除。

### 2.1 憑證列表

**顯示欄位：**
- 名稱（顯示名稱 + CN）
- 簽發者
- 到期日（含剩餘天數 / 已過期天數）
- 狀態（有效 / 已過期 / 已撤銷）
- 演算法（如 RSA 2048、EC P-256）
- 私鑰（有 / 無）

**警告提示：**
- ⚠ 即將到期（30 天內）— 橘色警告 icon
- ⚠ 無部署目標 — 灰色警告 icon（active 憑證未被任何部署使用）

**篩選功能：**
- 搜尋框：依名稱、CN 或 SAN 搜尋
- 狀態篩選：全部 / 有效 / 已過期 / 已撤銷
- 分頁

### 2.2 匯入憑證

**操作步驟：**
1. 點「匯入憑證」
2. 上傳或貼上憑證（支援拖曳多個檔案）
3. 上傳或貼上私鑰（選填）
4. 如為 PFX / JKS 格式，輸入檔案密碼
5. 填寫顯示名稱、標籤、備註（選填）
6. 按「匯入」

**支援格式：**
- PEM（.pem, .crt, .cer）
- DER（.der, .cer）
- PFX / PKCS#12（.pfx, .p12）— 需要密碼
- JKS（.jks）— 需要密碼
- P7B / PKCS#7（.p7b）

**自動處理：**
- 自動辨識格式
- 自動合併多個檔案為憑證鏈
- 自動偵測加密私鑰並提示輸入密碼

### 2.3 檢視憑證詳情

點擊列表中的 👁 圖示，顯示：

**基本資訊：**
- CN、簽發者、SAN、序號
- 演算法、有效期間、指紋
- 狀態、來源、建立者、建立時間
- 標籤、備註

**憑證鏈：**
- 自動載入並顯示完整鏈
- 標示角色（伺服器 / 中繼 / 根）

**下載：**
- PEM（可選擇是否包含私鑰 — 適用 HAProxy）
- KEY（僅私鑰）
- DER
- PFX（需設定密碼）
- JKS（需設定密碼）

**部署目標：**
- 在詳情頁底部顯示該憑證的所有部署目標
- 可直接新增 / 編輯 / 刪除部署目標

### 2.4 編輯憑證

點擊列表中的 ✏ 圖示：

**可編輯欄位：**
- 顯示名稱
- 標籤
- 備註
- 補上傳私鑰（如果匯入時沒有）

### 2.5 刪除憑證

點擊列表中的 🗑 圖示，會出現確認對話框。刪除後無法復原。

## 3. CSR 管理

管理憑證簽署請求（CSR），用於向 CA 申請憑證。

### 3.1 CSR 列表

**顯示欄位：**
- CN（通用名稱）
- SAN
- 演算法（如 RSA 2048）
- 狀態（待簽 / 已簽）
- 建立日期

**篩選功能：**
- 搜尋框：依 CN 或 SAN 搜尋
- 狀態篩選：全部 / 待簽 / 已簽
- 分頁

### 3.2 產生 CSR

點「產生 CSR」按鈕，欄位同工具箱的「產生 CSR」。

差異：工具箱產生後只下載檔案；CSR 管理產生後會**存入系統**，方便後續追蹤。

### 3.3 CSR 詳情

點 👁 圖示，顯示：
- 完整主體資訊
- 金鑰資訊（類型、位元數、是否加密）
- 狀態
- 關聯的憑證（如果已簽回）
- CSR 內容（可複製）
- 下載按鈕

### 3.4 CSR 簽回流程

1. 在 CSR 管理產生 CSR
2. 下載 CSR 檔案，送給 CA 簽發
3. CA 回傳簽發的憑證
4. 到「憑證管理」匯入該憑證
5. 系統自動關聯 CSR 與憑證（狀態變為「已簽」）

## 4. 部署目標

管理憑證要部署到哪些主機、哪些服務。搭配 hycert-agent 自動部署。

### 4.1 部署列表

**顯示資訊：**
- 目標主機
- 服務類型（nginx / apache / haproxy / hyproxy / tomcat / iis / other）
- 連接埠
- 管理狀態（使用中 / 停用中）
- 部署狀態（待部署 / 已部署 / 部署失敗）
- 部署路徑詳情（OS、憑證路徑、私鑰路徑、重載指令）
- 憑證名稱 + 指紋
- 最後部署時間

**警告提示：**
- ⚠ 未指定 Agent
- ⚠ Agent 不存在 / 已停用 / 離線
- ⚠ Token 已停用
- ⚠ 標籤不匹配（部署 label ≠ Token label）

**篩選功能：**
- 搜尋框：依主機或服務搜尋
- 部署狀態篩選：全部 / 已部署 / 待部署 / 失敗
- 分頁

### 4.2 新增部署目標

點「新增部署」按鈕：

| 欄位 | 說明 | 必填 |
|------|------|------|
| 憑證 | 選擇要部署的憑證（搜尋式選單） | ✓ |
| Agent | 選擇目標主機的 Agent（搜尋式選單，顯示在線狀態） | 建議 |
| 標籤 | 選擇分群標籤（從 Token 的 label 列表） | — |
| 目標主機 | IP 或主機名稱（選擇 Agent 後自動帶入） | ✓ |
| 服務類型 | nginx / apache / haproxy / hyproxy / tomcat / iis / other | ✓ |
| 連接埠 | 服務埠號（如 443） | — |
| 作業系統 | Linux / Windows | ✓ |
| 憑證路徑 | 目標主機上的憑證存放路徑 | ✓ |
| 私鑰路徑 | 目標主機上的私鑰存放路徑（PEM 類型） | 視服務 |
| Keystore 密碼 | JKS / PFX 的密碼（選 tomcat / iis 時出現） | 視服務 |
| Key 別名 | JKS 的 key alias（選 tomcat 時出現） | 視服務 |
| 重載指令 | 部署後執行的服務重啟指令 | 建議 |
| 備註 | 自由備註 | — |

**各服務類型的設定範例：**

| 服務 | 憑證路徑 | 私鑰路徑 | 重載指令 |
|------|---------|---------|---------|
| nginx | `/etc/nginx/ssl/server.pem` | `/etc/nginx/ssl/server.key` | `nginx -s reload` |
| apache | `/etc/httpd/ssl/cert.pem` | `/etc/httpd/ssl/key.pem` | `systemctl reload httpd` |
| haproxy | `/etc/haproxy/ssl/cert.pem` | （不需要，合併在憑證內） | `systemctl reload haproxy` |
| hyproxy | `/hyproxy/ssl/cert.pem` | `/hyproxy/ssl/key.pem` | `systemctl restart hyproxy` |
| tomcat | `D:\server.jks` | （不需要，包含在 JKS） | `Restart-Service Tomcat8` |

### 4.3 編輯部署目標

點 ✏ 圖示，可修改所有欄位。包含管理狀態（使用中 / 停用中）。

- **使用中**：Agent 會拉取此部署目標
- **停用中**：Agent 不會拉取，但資料保留

### 4.4 刪除部署目標

點 🗑 圖示，確認後刪除（軟刪除，從列表消失）。

### 4.5 部署歷程

點 ▼ 展開圖示，查看該部署目標的歷次部署記錄：

| 欄位 | 說明 |
|------|------|
| 時間 | 部署時間 |
| 動作 | deploy / rollback |
| 狀態 | 成功 / 失敗 |
| 指紋 | 部署的憑證指紋（前 19 字元） |
| 耗時 | 部署花費時間（毫秒） |
| 錯誤 | 失敗時的錯誤訊息 |

### 4.6 自動部署流程

```
1. PM 在 UI 匯入憑證（或更新憑證）
2. PM 建立部署目標（綁定 Agent + 服務路徑 + 標籤）
3. Agent 自動輪詢偵測到新的憑證指紋
4. Agent 自動下載 → 備份舊檔 → 寫入新檔 → 重啟服務
5. Agent 回報部署結果到 UI
6. PM 在 UI 查看部署狀態和歷程
```

### 4.7 標籤（Label）與部署過濾

部署目標可設定標籤，用於分群管控：

- Token 有 label + Deployment 有相同 label → Agent 拉得到
- Token 有 label + Deployment label 不同 → Agent 拉不到
- 無 label（空）→ 視為萬用，匹配所有

如果標籤不匹配，部署列表會顯示 ⚠ 警告提示。

## 5. Agent 管理

查看所有已註冊的部署代理程式（Agent）。Agent 安裝在客戶主機上，負責自動部署憑證。

### 5.1 Agent 列表

**顯示欄位：**

| 欄位 | 說明 |
|------|------|
| 狀態 | 在線（綠色）/ 離線（灰色）/ 已停用（紅色） |
| 名稱 | Agent 安裝時設定的顯示名稱 |
| 主機名稱 | 作業系統的 hostname |
| IP 位址 | 主機的網路 IP（自動偵測） |
| Token | 使用的 Token 名稱（⚠ 表示 Token 已停用或過期） |
| 作業系統 | Linux / Windows |
| 版本 | Agent 的版本號 |
| 輪詢間隔 | 多久檢查一次（如 1h） |
| 最後上線 | Agent 最後一次回報時間 |
| 操作 | 停用 / 啟用 |

**篩選功能：**
- 搜尋框：依名稱、主機名稱或 IP 搜尋
- 狀態篩選：全部 / 在線 / 離線

**在線判斷：**
- 最後上線時間在「輪詢間隔 × 2」以內 = 在線

### 5.2 停用 / 啟用 Agent

- **停用**：Agent 無法拉取部署目標（API 回 403），server log 記錄嘗試
- **啟用**：恢復正常拉取

### 5.3 Agent 安裝方式

Agent 需要由系統管理員在客戶主機上安裝。安裝後會自動向系統註冊，並出現在 Agent 列表中。

- **Linux**：使用 `deploy.sh` 安裝（詳見 install-linux.md）
- **Windows**：雙擊 `install.bat` 安裝（詳見 install-windows.md）

## 6. Token 管理

管理 Agent 的認證憑據（Token）。Token 用於 Agent 連線 API 時的身份驗證。

### 6.1 Token 列表

**顯示欄位：**

| 欄位 | 說明 |
|------|------|
| 名稱 | Token 的顯示名稱 |
| Token 前綴 | `hycert_agt_xxxxxxxx...`（辨識用） |
| 標籤 | 分群標籤（用於過濾部署目標） |
| Agent | 使用此 Token 的 Agent 數量（可展開看明細） |
| 狀態 | 啟用中（綠色）/ 已停用（紅色） |
| 最後使用 | Agent 最後一次用此 Token 認證的時間 |
| 到期日 | 過期後自動失效（空 = 永不過期） |
| 操作 | 顯示 / 編輯 / 停用 / 刪除 |

### 6.2 建立 Token

點「建立 Token」：

| 欄位 | 說明 | 必填 |
|------|------|------|
| 名稱 | Token 的顯示名稱（如 `token-web-server`） | ✓ |
| 標籤 | 分群標籤（如客戶代碼、環境名稱） | — |
| 到期日 | 過期日期（空 = 永不過期） | — |

建立後會顯示 Token 明文，請妥善保管。日後可透過「顯示」按鈕再次查看。

### 6.3 顯示 Token

點 👁 圖示，解密並顯示 Token 明文。可複製後用於安裝 Agent。

> 注意：系統更新前建立的 Token 可能無法顯示（缺少加密存儲），需建立新的替代。

### 6.4 編輯 Token

點 ✏ 圖示，可修改：
- 名稱
- 標籤

### 6.5 停用 Token

點 🛡 圖示，Token 狀態變為「已停用」。影響：
- **所有使用此 Token 的 Agent 都無法認證**
- Agent 會收到 401 錯誤
- 已停用的 Token 無法恢復，需建立新的

### 6.6 刪除 Token

點 🗑 圖示，永久刪除。限制：
- **有 Agent 綁定的 Token 無法刪除**（不管 Agent 是否停用）
- 需先讓所有 Agent 改用其他 Token，才能刪除

### 6.7 標籤（Label）使用方式

標籤用於分群管控，常見用法：

| 分群方式 | 標籤範例 |
|---------|---------|
| 按客戶 | `customer-A`、`mhchcm` |
| 按環境 | `prod`、`staging` |
| 按機房 | `taipei-dc`、`kaohsiung-dc` |

同一標籤的 Token 可在多台主機安裝時**自動重用**（安裝腳本會查詢 by-label API）。

## 三層管控總覽

| 層級 | 操作 | 影響範圍 |
|------|------|---------|
| Token 停用 | 該 Token 下所有 Agent 無法認證 | 最大（整個標籤群組） |
| Agent 停用 | 該 Agent 無法拉取部署目標 | 中（單台主機） |
| 部署目標停用 | 該筆部署不再執行 | 最小（單個服務） |
| 部署目標刪除 | 從列表消失 | 最小（單個服務） |

## 常見操作流程

### 流程一：新客戶憑證上線

1. 到客戶主機安裝 Agent（輸入標籤 = 客戶代碼）
2. 收到客戶的憑證檔案
3. 到「工具箱 → 檢核」驗證憑證正確
4. 到「憑證管理 → 匯入」匯入憑證
5. 到「部署目標 → 新增」建立部署設定（選 Agent + 標籤 + 服務路徑）
6. 等待 Agent 自動部署（或到主機手動執行 `hycert-agent run`）
7. 在「部署目標」查看部署狀態

### 流程二：憑證續約

1. 收到新的憑證檔案（同一個域名）
2. 匯入新憑證
3. 到現有的部署目標，編輯「憑證」欄位改為新憑證
4. Agent 偵測到指紋不同，自動重新部署

### 流程三：透過 CSR 申請憑證

1. 到「CSR 管理 → 產生 CSR」
2. 下載 CSR 檔案，送交 CA 簽發
3. 收到 CA 簽發的憑證後，到「憑證管理 → 匯入」
4. 系統自動關聯 CSR（狀態變為「已簽」）
5. 建立部署目標

### 流程四：停止某客戶的自動部署

- **停止整個客戶**：到 Token 管理 → 停用該客戶的 Token
- **停止某台主機**：到 Agent 管理 → 停用該 Agent
- **停止某個服務**：到部署目標 → 編輯狀態改為「停用中」

## 注意事項

1. **私鑰安全**：匯入的私鑰在系統內以 Tink 加密儲存，不會以明文存在資料庫
2. **傳輸安全**：Agent 與 Server 之間建議使用 HTTPS，憑證（含私鑰）傳輸才有加密保護
3. **備份**：Agent 部署前會自動備份現有的憑證檔案
4. **權限**：Agent 需要有目標路徑的寫入權限和服務重啟權限
5. **PFX / JKS 密碼**：匯入時的密碼（解開原始檔案用）和部署時的密碼（產生 keystore 用）可以不同
6. **Token 保管**：Token 建立後可透過「顯示」按鈕再次查看。安裝腳本輸入相同標籤時會自動重用 Token
7. **標籤匹配**：部署目標的標籤必須與 Token 的標籤一致，否則 Agent 拉不到部署目標（警告提示會顯示）
